HSTP FSP reconciliation

Upload the FSP reconciliation file + the original HOPE payment file. The output is aligned to HOPE rows and mapped by Payment ID. Extra FSP rows (not found in HOPE) are appended at the bottom.

Standard output fields

  • payment_id
  • individual_id
  • reference_id
  • Biometric referance
  • entitlement_quantity
  • delivered_quantity
  • delivery_date
  • reason_for_unsuccessful_payment







Join is performed using the selected columns. Output rows follow HOPE, and unmatched FSP rows are added at the end.

Rule types: copy (direct copy) and conditional_copy (copy when a condition matches). Conditions can reference HOPE or FSP columns because the files are joined by Payment ID first.
